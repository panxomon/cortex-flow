# Skill — ADR Writer

Objective:
Capture consequential decisions made during analysis or execution.

---

# CREATE AN ADR WHEN

* a divergence emerges
* a meaningful tradeoff exists
* the design changes
* relevant technical debt is introduced
* critical ambiguity surfaces

---

# FORMAT

Use:
`.ai/templates/adr-template.md`

---

# RULES

* Keep ADRs small and focused
* Prioritize clarity
* Record real consequences, not hopes
* Avoid bureaucratic boilerplate

---

# OUTPUT

`definitions/adr/`

ADRs are written in the operator's working language (periphery), not in English (unless that is the working language).
